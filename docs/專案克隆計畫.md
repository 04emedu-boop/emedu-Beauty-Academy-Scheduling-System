# 從 GitHub Clone emedu-Beauty-Academy-Scheduling-System 專案

## 問題描述

使用者希望從 GitHub 將 `emedu-Beauty-Academy-Scheduling-System` 倉庫 clone 到 `d:\projects`。

**當前狀態:**
- ✅ GitHub CLI v2.83.1 已安裝並在系統 PATH 中
- ❌ Git 無法從 PowerShell 訪問 (不在 PATH 中)
- ❌ Git 可能尚未安裝在系統上

## 建議解決方案

由於無法從當前 PowerShell 會話訪問 Git,我們有兩個選項:

### 選項 A: 使用 GitHub CLI (推薦)
GitHub CLI 包含 Git 功能且已安裝。但需要重啟終端機才能使用。

**步驟:**
1. 請使用者重啟終端機/VS Code
2. 使用 `gh auth login` 驗證 GitHub 帳號
3. 使用 `gh repo clone 04EMEDU/emedu-Beauty-Academy-Scheduling-System` clone 倉庫

### 選項 B: 安裝 Git 並 Clone
如果 Git 未安裝,需要先安裝。

**步驟:**
1. 使用 `winget install --id Git.Git -e` 安裝 Git
2. 重啟終端機
3. 使用 `git clone https://github.com/04EMEDU/emedu-Beauty-Academy-Scheduling-System.git` clone

> [!IMPORTANT]
> 兩種選項都需要重啟終端機,因為新安裝的工具需要加入 PATH 環境變數,這只在新的終端機會話中生效。

## 建議方法

**請使用者重啟終端機/VS Code**,然後使用 GitHub CLI clone 倉庫,因為它已經安裝完成。

## 驗證計畫

### 手動驗證
1. 使用者重啟終端機/VS Code
2. 執行 `gh auth login` 並驗證 GitHub 帳號
3. 在 `d:\projects` 執行 `gh repo clone 04EMEDU/emedu-Beauty-Academy-Scheduling-System`
4. 驗證倉庫資料夾存在於 `d:\projects\emedu-Beauty-Academy-Scheduling-System`
5. 驗證 clone 的目錄中有檔案
